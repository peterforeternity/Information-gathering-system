# 数据库设计说明

## 1. 概述

- **数据库类型**：SQLite（通过 `better-sqlite3` 驱动，同步 API、预编译语句）。
- **数据库文件**：`${DATA_DIR}/app.db`，`DATA_DIR` 默认为项目下 `data/` 目录，可通过环境变量指向持久化卷（如 Docker 数据卷）。
- **连接参数**：
  - `PRAGMA journal_mode = WAL`（Write-Ahead Logging，提升并发读写性能）；
  - `PRAGMA foreign_keys = ON`（启用外键约束，保证级联删除生效）。
- **建表策略**：应用启动时执行 `CREATE TABLE IF NOT EXISTS`，幂等初始化，重启后数据不丢失。

## 2. E-R 关系

```
users (独立)                students 1 ──────< N grades
                            一个学生可有多条成绩记录；删除学生级联删除其成绩。
                            (studentId, subject) 唯一：同一学生同一科目仅一条记录。
```

## 3. 表结构

### 3.1 users（用户）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | TEXT | PRIMARY KEY | 用户唯一标识（UUID v4） |
| username | TEXT | UNIQUE, NOT NULL | 用户名（≥3 字符，业务层校验） |
| passwordHash | TEXT | NOT NULL | scrypt 哈希（hex，64 字节） |
| salt | TEXT | NOT NULL | 随机盐（hex，16 字节） |
| role | TEXT | NOT NULL | 角色：`admin` / `student` |

```sql
CREATE TABLE IF NOT EXISTS users (
  id TEXT PRIMARY KEY,
  username TEXT UNIQUE NOT NULL,
  passwordHash TEXT NOT NULL,
  salt TEXT NOT NULL,
  role TEXT NOT NULL
);
```

### 3.2 students（学生）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | TEXT | PRIMARY KEY | 学生唯一标识（UUID v4） |
| studentNo | TEXT | UNIQUE, NOT NULL | 学号（全局唯一） |
| name | TEXT | NOT NULL | 姓名 |
| gender | TEXT | NOT NULL | 性别：`男` / `女` |
| className | TEXT | NOT NULL | 班级 |

```sql
CREATE TABLE IF NOT EXISTS students (
  id TEXT PRIMARY KEY,
  studentNo TEXT UNIQUE NOT NULL,
  name TEXT NOT NULL,
  gender TEXT NOT NULL,
  className TEXT NOT NULL
);
```

### 3.3 grades（成绩）

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | TEXT | PRIMARY KEY | 成绩记录唯一标识（UUID v4） |
| studentId | TEXT | NOT NULL, FK → students(id) ON DELETE CASCADE | 所属学生 |
| subject | TEXT | NOT NULL | 科目名称 |
| score | REAL | NOT NULL | 分数（业务层限制 0–100） |
| — | — | UNIQUE(studentId, subject) | 同一学生同一科目仅一条 |

```sql
CREATE TABLE IF NOT EXISTS grades (
  id TEXT PRIMARY KEY,
  studentId TEXT NOT NULL,
  subject TEXT NOT NULL,
  score REAL NOT NULL,
  UNIQUE(studentId, subject),
  FOREIGN KEY(studentId) REFERENCES students(id) ON DELETE CASCADE
);
```

## 4. 约束与业务规则映射

| 业务规则 | 数据库实现 |
|----------|-----------|
| 用户名唯一 | `users.username UNIQUE` |
| 学号唯一 | `students.studentNo UNIQUE` |
| 同生同科成绩唯一/覆盖 | `grades UNIQUE(studentId, subject)` + `INSERT ... ON CONFLICT DO UPDATE` |
| 删除学生级联删除成绩 | 外键 `ON DELETE CASCADE` + `PRAGMA foreign_keys = ON` |
| 分数 0–100 | API 层校验（`Number.isFinite && 0..100`） |

## 5. 数据访问层（api/store.ts）

均为预编译语句，防注入、性能更优。核心方法：

| 方法 | SQL 语义 |
|------|----------|
| `getUserById / getUserByUsername` | `SELECT * FROM users WHERE ...` |
| `createUser` | `INSERT INTO users ...` |
| `getStudents` | `SELECT * FROM students ORDER BY studentNo` |
| `getStudentById / getStudentByNo` | `SELECT * FROM students WHERE ...` |
| `createStudent / updateStudent / deleteStudent` | `INSERT / UPDATE / DELETE` |
| `getGrades / findGrade` | `SELECT * FROM grades ...` |
| `upsertGrade` | `INSERT INTO grades ... ON CONFLICT(studentId, subject) DO UPDATE SET score = excluded.score` |

聚合成绩表（`GET /api/grades`）在应用层将 students 与 grades 组合为「每生一行 + 各科分数 + 平均分」结构。

## 6. 迁移与备份

- **从 JSON 迁移**：旧模型 `users/students/grades` 字段与新表一一对应，可读取旧 JSON 后按上表逐条 `INSERT` 完成迁移；本次环境无历史 JSON 数据，数据库以空表初始化。
- **备份**：直接复制 `app.db`（及 WAL/`-shm` 文件）或使用 `sqlite3 app.db ".backup"`。
- **持久化部署**：Docker 中将 `DATA_DIR` 指向数据卷（`-v gms-data:/app/data`），镜像内已设 `DATA_DIR=/app/data`。
