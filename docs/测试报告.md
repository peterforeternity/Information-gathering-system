## 4. 数据迁移验证

- 后端存储由 JSON 文件切换为 SQLite；`store.ts` 首次启动执行 `CREATE TABLE IF NOT EXISTS` 建立 `users/students/grades` 三表。
- 表字段与原 JSON 数据模型一一对应，唯一约束（用户名、学号、`(studentId, subject)`）与外键级联在数据库层生效（用例 6、8、13 验证）。
- 本次环境无历史 JSON 数据，数据库以空表初始化；写入/覆盖/级联删除均正确，未发现数据丢失或不一致。

- 原有功能全部保留并通过验证，前后端 API 契约保持一致。
- 构建与运行正常，13 项接口/功能用例全部通过（100%）。
