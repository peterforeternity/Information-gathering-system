# 成绩管理系统 技术文档

## 1. 概述

本系统是一个面向学校/班级场景的成绩管理系统，支持用户注册登录、学生信息维护、成绩录入与表格化展示。本文档描述系统在完成架构迁移后的技术方案：**前端由 React 迁移至 Vue 3，后端数据存储由本地 JSON 文件迁移至 SQL（SQLite）数据库**，原有功能完整保留。

## 2. 技术栈

| 层次 | 迁移前 | 迁移后 |
|------|--------|--------|
| 前端框架 | React 18 | **Vue 3（Composition API / `<script setup>`）** |
| 路由 | React Router 7 | **Vue Router 4** |
| 状态管理 | zustand | **Pinia** |
| 图标 | lucide-react | lucide-vue-next |
| 构建工具 | Vite（@vitejs/plugin-react） | Vite（**@vitejs/plugin-vue**） |
| 样式 | TailwindCSS | TailwindCSS（保持不变） |
| 后端 | Express 4 + TypeScript | Express 4 + TypeScript（保持不变） |
| 数据存储 | 本地 JSON 文件 | **SQLite（better-sqlite3）** |
| 鉴权 | scrypt 加盐哈希 + Token 会话 | 保持不变 |

## 3. 系统架构

```
┌──────────────────────────────┐        HTTP / JSON        ┌──────────────────────────────┐
│  前端 SPA（Vue 3 + Vite）       │  ───────────────────────▶ │  后端 API（Express）            │
│                              │   /api/auth /students     │                              │
│  Pinia(store) ─ Vue Router   │   /grades                 │  路由 → 鉴权中间件 → store      │
│  组件（.vue）─ fetch 封装      │  ◀─────────────────────── │                              │
└──────────────────────────────┘        JSON 响应           └──────────────┬───────────────┘
                                                                           │ better-sqlite3
                                                                           ▼
                                                                 ┌───────────────────┐
                                                                 │  SQLite: app.db     │
                                                                 │  users/students/    │
                                                                 │  grades             │
                                                                 └───────────────────┘
```

生产环境下由单个 Express 服务同时托管前端构建产物（`dist/`）与 API，非 `/api` 路由回退到 `index.html` 以支持前端路由。

## 4. 前端设计

### 4.1 目录结构

```
src/
  main.ts                入口：创建 app、注册 Pinia 与 Router、恢复会话后挂载
  App.vue                根组件：初始化时显示加载态，随后渲染 <router-view>
  router/index.ts        路由表与全局守卫（登录校验 / 管理员权限 / 游客限制）
  stores/auth.ts         Pinia 认证 store（login/register/logout/restore）
  utils/api.ts           统一 fetch 封装：自动携带 Bearer Token，统一错误处理
  types/index.ts         共享类型（Role/AuthUser/Student/GradeRow/GradeTable）
  lib/utils.ts           cn() 类名合并工具
  pages/
    Login.vue            登录 / 注册页
    Students.vue         学生管理（列表 + 增删改）
    Grades.vue           成绩管理（聚合表格 + 排序 + 录入 + 导出 Excel）
  components/
    Navbar.vue           顶部导航
    Modal.vue            通用弹窗容器（slot）
    StudentFormModal.vue 学生新增/编辑表单
    GradeFormModal.vue   成绩录入表单
```

### 4.2 关键实现

- **数据双向绑定**：表单控件统一使用 `v-model`（如登录、学生表单、成绩录入），配合 `ref` 响应式状态实现双向绑定。
- **组件化开发**：页面（pages）与可复用 UI（components）分离；`Modal.vue` 通过默认插槽承载不同表单内容；表单弹窗通过 props 接收 `open/editing/onSubmit`，通过 `emit('close')` 通知父级关闭。
- **路由管理**：`vue-router` 采用 `createWebHistory`，在 `beforeEach` 全局守卫中统一处理：
  - `requiresAuth`：未登录跳转 `/login`；
  - `adminOnly`：非管理员跳转 `/grades`；
  - `guestOnly`：已登录用户访问登录页时跳转 `/grades`。
- **状态管理（Pinia）**：`useAuthStore` 使用 setup 语法定义，暴露 `user`、`initializing` 以及 `login/register/logout/restore`。应用启动时先 `restore()` 校验本地 Token，再挂载，确保路由守卫拿到正确登录态。
- **成绩表排序**：`Grades.vue` 使用 `computed` 派生 `sortedRows`，支持按学号/姓名/班级/任意科目/平均分升降序排序；空值恒排末尾。
- **Excel 导出**：沿用 `xlsx` 库，按当前排序导出为 `.xlsx`。

### 4.3 与后端通信

`utils/api.ts` 中的 `apiRequest<T>(path, { method, body })`：
- 自动附加 `Authorization: Bearer <token>`（token 存于 `localStorage`，键 `gms_token`）；
- 统一解析 JSON，对 `!res.ok` 或 `success === false` 抛出 `ApiError(message, status)`；
- 开发环境通过 Vite 代理将 `/api` 转发至 `http://localhost:3001`。

## 5. 后端设计

### 5.1 API 接口

所有响应统一为 `{ success: boolean, ... }` 结构。写操作（POST/PUT/DELETE 学生、成绩）需管理员权限。

| 方法 | 路径 | 权限 | 说明 |
|------|------|------|------|
| GET | `/api/health` | 公开 | 健康检查 |
| POST | `/api/auth/register` | 公开 | 注册（admin/student），返回 token |
| POST | `/api/auth/login` | 公开 | 登录，返回 token |
| POST | `/api/auth/logout` | 登录 | 登出，销毁会话 |
| GET | `/api/auth/me` | 登录 | 获取当前用户 |
| GET | `/api/students` | 登录 | 学生列表 |
| POST | `/api/students` | 管理员 | 新增学生（学号唯一） |
| PUT | `/api/students/:id` | 管理员 | 编辑学生 |
| DELETE | `/api/students/:id` | 管理员 | 删除学生（级联删除成绩） |
| GET | `/api/grades` | 登录 | 聚合成绩表：每生一行，含各科成绩与平均分 |
| GET | `/api/grades/subjects` | 登录 | 已有科目列表 |
| POST | `/api/grades` | 管理员 | 录入/更新成绩（同生同科覆盖；0–100 校验） |

### 5.2 鉴权

- 密码使用 Node `crypto.scryptSync` 加盐哈希（64 字节），校验时 `timingSafeEqual` 防时序攻击。
- 会话为内存态 `Map<token, userId>`，`requireAuth` 中间件注入 `req.user`，`requireAdmin` 限制管理员写操作。
- 注：会话为进程内存，服务重启需重新登录；业务数据已持久化到 SQLite，不受影响。

### 5.3 数据访问层

`api/store.ts` 基于 `better-sqlite3` 提供同步、预编译语句的 CRUD 封装，详见《数据库设计说明》。

## 6. 构建与运行

```bash
npm install
npm run dev      # 前端 5173 + 后端 3001（Vite 代理 /api）
npm run build    # vite build 生成 dist/
npm start        # NODE_ENV=production，由 Express 托管 dist 与 API
```

环境变量：`PORT`（默认 3001）、`NODE_ENV`、`DATA_DIR`（SQLite 文件目录，默认 `./data`）。

## 7. 迁移说明与兼容性

- **功能对等**：登录/注册、学生增删改查、成绩录入/覆盖、聚合表格排序、Excel 导出、角色权限控制均逐一迁移并通过验证。
- **接口不变**：前端仅替换框架，API 契约保持一致，后端无需为前端迁移改动。
- **数据迁移**：迁移后存储为 SQLite。表结构（见数据库设计说明）字段与原 JSON 模型一一对应（users/students/grades），`store.ts` 在首次启动时自动建表（`CREATE TABLE IF NOT EXISTS`）。若存在旧 JSON 数据，可按相同字段读取后逐条写入对应表完成迁移；本次迁移环境无历史 JSON 数据，数据库以空表初始化，验证无数据丢失。
- **性能与体验**：SQLite 采用 WAL 日志模式与预编译语句，读写更高效；学号/成绩唯一约束交由数据库保证；成绩「同生同科覆盖」由 `ON CONFLICT ... DO UPDATE` 原子完成。
